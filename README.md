# Viktoria

Viktoria — платформа для управления задачами с event-driven архитектурой, real-time синхронизацией и системой автоматизации.

Проект разработан в рамках хакатона как современный аналог Jira/Trello, ориентированный не только на хранение задач, но и на обработку событий внутри команды.

---

## Идея проекта

Большинство kanban-систем работают как обычный CRUD:
пользователь изменяет задачу → данные обновляются в базе.

В Viktoria любое действие становится событием:

* создание задачи,
* смена статуса,
* перемещение между колонками,
* изменение дедлайна,
* назначение исполнителя.

События проходят через очередь RabbitMQ, обрабатываются automation engine и синхронизируются между всеми пользователями в реальном времени.

Это позволяет строить масштабируемую систему автоматизации процессов, а не просто доску задач.

---

## Возможности системы

### Kanban-доска

* To Do / In Progress / Done
* кастомные колонки
* drag & drop задач
* теги и приоритеты
* дедлайны
* фильтрация и поиск

### Real-time синхронизация

Изменения мгновенно отображаются у всех участников через WebSocket:

* создание задач,
* обновления,
* перемещения,
* уведомления,
* изменения статусов.

### Event-driven обработка

Внутри системы реализован pipeline обработки событий:

```text
API → RabbitMQ → обработка → automation → WebSocket
```

События:

* валидируются,
* дедуплицируются,
* обогащаются метаданными,
* логируются,
* отправляются подписчикам.

### Автоматизация

Система поддерживает автоматические реакции на события:

* уведомления,
* автоматическое изменение статусов,
* флаги по тегам,
* обработка правил workflow.

### AI-помощник

AI помогает при работе с задачами:

* генерация описаний,
* task grooming,
* уточнение требований,
* помощь при создании задач.

## Архитектура

```text
Frontend (Next.js)
        ↓
FastAPI Backend
        ↓
RabbitMQ Event Bus
        ↓
Automation Engine
        ↓
PostgreSQL + WebSocket Sync
```

Проект разделён на независимые backend-модули:

* auth,
* workspace,
* board,
* tasks,
* events,
* automation,
* notifications,
* analytics.

Такой подход упрощает масштабирование и развитие платформы.
