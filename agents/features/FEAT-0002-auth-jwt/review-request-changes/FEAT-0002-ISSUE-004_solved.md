# FEAT-0002-ISSUE-004: Решено
## Что исправлено
`logout` теперь вызывает `decode_token(refresh_token)` перед добавлением токена в blocklist. Если токен невалидный, `HTTPException` перехватывается и функция возвращает управление без действий (идемпотентное поведение). Только валидные токены попадают в blocklist.
